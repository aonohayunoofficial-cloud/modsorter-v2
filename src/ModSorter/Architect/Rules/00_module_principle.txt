# ===== 第0層: モジュール構成の大原則(全ジャンル共通・最優先) =====
# 生成するのは「完成品」ではなく、他モジュールと接続できる単機能の骨格モジュールである。
# どのジャンルでも、機能ブロック単体で終わらせず、必ず「入力口」と「出力口」(ガワ)を含めること。

# --- 必ず3要素を含める ---
# 1. 核(コア): そのモジュールの主機能ブロック(例: 加工なら millstone、動力源なら water_wheel)。
# 2. 入力口(インプット): 核に「外から何かを受け取る」ための接続部。
# 3. 出力口(アウトプット): 核から「外へ何かを渡す」ための接続部。
# 核1ブロックだけで終わってはいけない。入力口・出力口のどちらかでも欠けてはいけない。

# --- ジャンル別の「入力口/出力口」の意味 ---
# [動力源]
#   入力口: 不要(動力を自分で生む)。代わりに動力を生む条件部があればよい。
#   出力口: 生んだ回転を外へ渡す shaft/cogwheel を、回転軸に合わせて1つ付ける。
# [動力伝達・分配]
#   入力口: 回転を受け取る端の shaft/cogwheel。
#   出力口: 回転を次へ渡す端の shaft/cogwheel。直線または分岐の両端を必ず作る。
# [動力制御]
#   入力口: 制御対象の回転を受ける側の shaft。
#   出力口: 制御後の回転を出す側の shaft。clutch/gearshift の前後を必ず作る。
# [加工]
#   入力口: 核に動力を入れる shaft/cogwheel(機械ごとの正しい面に)。
#           素材を受ける口(depot や上面投入位置)も用意する。
#   出力口: 加工物を出す funnel + その先の depot/chest。
# [保管]
#   入力口: アイテムを入れる funnel。
#   出力口: アイテムを出す funnel。入れる側と出す側の両方を付ける。
# [流体]
#   入力口: 流体を受ける fluid_pipe の端。
#   出力口: 流体を出す fluid_pipe の端。
# [可動・構造]
#   入力口: 動かすための shaft(piston/bearing 等の動力入力)。
#   出力口: 動かす対象(接着するブロック)の起点を1つ示す。
# [計測・表示]
#   入力口: 計測する回転を受ける shaft(ラインの途中に挟む形)。
#   出力口: 同じラインを通す反対側の shaft。
# [レッドストーン連動]
#   入力口: 信号を受ける位置(lever 等の信号源 or 入力面)。
#   出力口: 制御される対象(clutch/gearshift など)を1つ含める。

# --- 接続口の作り方(重要) ---
# - 入力口/出力口の shaft・cogwheel・funnel・pipe は、核に物理的に隣接させ、向き/軸を合わせる。
# - 「ここから外へ繋がる」端であることを示すため、口の shaft/cogwheel は1ブロック分だけ伸ばす。
# - 入出力の位置の目印として、lime_wool(出力)や magenta_wool(入力)を口の隣に1つ置いてよい。

# --- やってはいけないこと ---
# - 核となる機能ブロックを1個だけ置いて終わる。
# - 入力口だけ、または出力口だけしか作らない。
# - 動力源・加工など、外と繋ぐ必要があるのに接続口を一切付けない。
